플러그인 선택 마지막 선택이 서버에 저장되어 기본값이 됩니다

플러그인별 연산 장치(CPU/GPU)는 「플러그인 옵션」 탭에서 플러그인마다 따로 정합니다. 비워두면 각 플러그인의 기본값을 씁니다.

번역 프롬프트 LLM 시스템 프롬프트를 고르거나 직접 편집

빌트인을 편집해 같은 이름으로 저장하면 그 이름의 커스텀 프리셋이 덮어씁니다. 새 이름으로 저장하면 복제됩니다. 커스텀을 삭제하면 빌트인으로 되돌아갑니다.

플러그인 옵션 선택된 플러그인의 옵션 (예: 번역 모델 태그)

플러그인 설치 패키지 + 가중치 원클릭 설치

동작 확장 동작 설정

이미지의 짧은 변이 이 값보다 작으면 아이콘·배너로 보고 번역하지 않습니다. 확장이 접속 시 이 값을 받아 적용합니다. 0 = 모든 이미지 번역.

켜면 탐지 영역·bounds·인식/번역 결과를 로그에 남깁니다(진단용, 재시작 없이 즉시 적용). 끄면 개수·시간 요약만.

GPU 락 밖에서 한 번에 번역할 이미지 수(재시작 없이 즉시 적용). ollama의 OLLAMA_NUM_PARALLEL과 맞추세요 — 둘 중 낮은 값이 실제 병렬도이고, 이 값만 높이면 요청이 큐에 밀려 타임아웃·폴백이 납니다.

로컬 torch 엔진(탐지·인식)을 이 시간(분) 동안 안 쓰면 VRAM에서 내려, 작업 사이에 GPU를 붙잡고 있지 않게 합니다(ollama 번역기는 별도 프로세스라 무관 — 그쪽은 OLLAMA_KEEP_ALIVE가 관리). 0 = 안 내림(상주 유지).

torch 엔진(탐지·인식)을 CPU로 돌릴지 GPU로 돌릴지. GPU면 패스스루된 장치로 벤더(AMD ROCm / NVIDIA CUDA)를 자동 판별해 맞는 torch를 깝니다. 설치 전에 정하세요 — 바꾸면 엔진 재설치(또는 /plugins 리셋 후 재설치)해야 적용됩니다. GPU 패스스루는 docker-compose 오버라이드로 따로 설정.
